# AegisTune 1.0.25.0

## Highlights

- Turned the shell into a guided maintenance cockpit instead of a generic utility menu.
- Added in-app update feed checks for packaged and portable builds, with one shared stable channel.
- Added a built-in release notes viewer so operators can review what changed before opening the update source.

## Guided workflow changes

- Home now surfaces update posture, next-step module guidance, and a cleaner operator-first summary.
- Drivers & Firmware now makes the selected device and next action obvious after an audit instead of leaving the queue visually dead.
- Apps & Uninstall now supports guided uninstall, automatic rescan, and confirmed leftover cleanup with quarantine-first handling.
- Repair & Recovery can now hand app-related cleanup issues directly into the app review flow.
- Safety & Undo now exposes restore points, registry rollback entries, startup changes, and driver-install history as one review surface.
- Audio & Sound was added as a dedicated module for endpoint volume, mute, review thresholds, and quick access to native Windows sound tools.

## Fixes

- Fixed the portable executable branding path so the published `.exe` carries the app icon and product metadata.
- Fixed update-feed staging so GitHub Pages and Cloudflare R2 can publish one consistent static update site.
- Fixed multiple UI clarity gaps where the user could complete a scan or audit but still not know what to click next.

## Operator notes

- The packaged and portable builds now point to the same update manifest, but packaged installs still rely on `AegisTune.appinstaller`.
- The current `MSIX` signing lane still uses the local `CN=ichiphost` certificate. Replace it with a trusted public code-signing certificate before broad external rollout.
- Release feed publishing is now scriptable, but `build-portable` and `build-msix` should still run sequentially in the same workspace.
